🚀 DropShip IL – מדריך הפעלה מלא

מדריך שלב-אחר-שלב להפעלת האפליקציה, חיבור חנויות, ומה צריך לעשות כדי להתחיל למכור

📑 תוכן עניינים

1. מבנה הפרויקט 2. הפעלת האפליקציה 3. חיבור חנויות 4. תהליך העבודה היומי 5. מה חסר לך לעשות עכשיו 6. מעבר ל-Production (אתר אמיתי) 7. פתרון בעיות נפוצות

🏗 מבנה הפרויקט

dropship-app/
  ├── backend/          # שרת Node.js
  │   ├── server.js         # נקודת הכניסה, port 3001
  │   ├── routes/
  │   │   ├── ebay.js       # eBay OAuth + API
  │   │   ├── shopify.js    # Shopify OAuth + API
  │   │   ├── amazon.js     # Amazon SP-API OAuth
  │   │   ├── etsy.js       # Etsy OAuth (PKCE)
  │   │   ├── woocommerce.js # WooCommerce REST API
  │   │   ├── products.js   # ניהול מוצרים (JSON file)
  │   │   └── orders.js     # ניהול הזמנות (in-memory)
  │   ├── data/             # products.json נשמר כאן
  │   └── .env              # API keys (לא מועלה ל-git!)
  └── frontend/         # React + Vite, port 5173
      └── src/components/
          ├── Layout        # סיידבר ניווט
          ├── Dashboard     # לוח בקרה
          ├── ProductSearch # חיפוש מוצרים מספקים
          ├── MyProducts    # ניהול מוצרים שמורים
          ├── StoreConnections # חיבור 5 חנויות
          ├── Orders        # ניהול הזמנות
          └── Settings      # הגדרות מחירים

Stack טכנולוגי

BackendNode.js + Express + Axios
FrontendReact 18 + Vite + Tailwind CSS
שמירת מוצריםקובץ JSON מקומי (data/products.json)
שמירת הזמנותin-memory (מתאפס בהפעלה מחדש)
AuthenticationOAuth 2.0 לכל הפלטפורמות

▶ הפעלת האפליקציה

1

פתח טרמינל ראשון – Backend

cd "Desktop/פתיחת חנות דרופ שיפינג/dropship-app/backend"
npm install
npm run dev

תראה: 🚀 שרת דרופ שיפינג רץ על פורט 3001

2

פתח טרמינל שני – Frontend

cd "Desktop/פתיחת חנות דרופ שיפינג/dropship-app/frontend"
npm install
npm run dev

תראה: VITE ready → http://localhost:5173

3

פתח בדפדפן

כנס לכתובת: http://localhost:5173

💡 ה-Vite dev server מועבר אוטומטית ל-backend (proxy). לכן /api/... עובד ישירות מה-frontend.
4

הגדר קובץ .env (אופציונלי לבינתיים)

העתק את .env.example לקובץ .env והוסף API keys אמיתיים לאחר שתרשם לפלטפורמות.

cp .env.example .env
⚠️ ללא API keys אמיתיים, השתמש ב"מצב הדגמה" – הכל עובד אבל לא מתחבר לחנויות אמיתיות.

🔗 חיבור חנויות – מדריך לכל פלטפורמה

🛒

eBay

פלטפורמת המכירות הגדולה בישראל ובעולם

OAuth 2.0
  1. הרשם ב-developer.ebay.com
  2. צור Application → Get Keys
  3. הוסף Redirect URI: http://localhost:3001/api/ebay/callback
  4. הוסף ל-.env: EBAY_CLIENT_ID, EBAY_CLIENT_SECRET
developer.ebay.com
🏪

Shopify

פלטפורמת חנות אונליין מקצועית

OAuth 2.0
  1. הרשם ב-partners.shopify.com
  2. Apps → Create app → Public app
  3. הגדר Redirect URI לפי ה-.env
  4. הוסף Scopes: read/write products & orders
partners.shopify.com
📦

Amazon

הפלטפורמה הגדולה בעולם – קהל ענק

SP-API OAuth
  1. חשבון Seller Central פעיל
  2. Apps & Services → Develop Apps
  3. Register as Developer
  4. צור LWA Application → קבל Client ID/Secret
sellercentral.amazon.com/apps/manage
🎨

Etsy

מוצרים ייחודיים, קהל קנייה ממוקד

OAuth 2.0 + PKCE
  1. הרשם ב-etsy.com/developers
  2. Create a New App
  3. הגדר Callback URL
  4. הוסף ל-.env: ETSY_CLIENT_ID (ה-Keystring)
etsy.com/developers/register
🔌

WooCommerce

חנות WordPress – שליטה מלאה

Consumer Key/Secret
  1. לוח ניהול WordPress שלך
  2. WooCommerce → הגדרות → מתקדם → REST API
  3. לחץ "הוסף מפתח" → בחר Read/Write
  4. הדבק את המפתחות ישירות בממשק
אין צורך ב-.env

🔄 תהליך העבודה היומי

1

חיפוש מוצרים לשונית "חיפוש מוצרים"

חפש מוצרים מספקים (כמו AliExpress). לכל מוצר תראה:

  • מחיר עלות מהספק (USD)
  • מחיר מכירה מוצע + רווח משוער (ILS)
  • דירוג, ביקורות, ומספר הזמנות

לחץ "+ הוסף למוצרים שלי" לשמירה.

2

עריכה ופרסום לשונית "המוצרים שלי"

  • ערוך – שנה שם, תיאור, ואחוז רווח
  • פרסם – בחר חנויות לפרסום (eBay, Shopify, Amazon, Etsy, WooCommerce)
  • המוצר יפורסם בכל החנויות שבחרת
3

ניהול הזמנות לשונית "הזמנות"

  • הזמנה חדשה מגיעה → סטטוס ממתין
  • הזמן מהספק → לחץ "הכנס לטיפול" → בטיפול
  • אחרי שהספק שלח → לחץ "סמן כנשלח" + הכנס מספר מעקב
  • חבילה הגיעה → הושלם

✅ מה חסר לך לעשות עכשיו

רשימת משימות לפי סדר עדיפות – מה שחובה לפני שמתחילים למכור:

🔴
חובה הרשמה לפחות לפלטפורמה אחת
בחר את הפלטפורמה שהכי מתאימה לך ורשום מוכר:
eBay ישראל – הכי פשוט להתחיל, קהל ישראלי גדול | Etsy – אם המוצרים שלך ייחודיים | Shopify – אם רוצה חנות עצמאית עם מיתוג
🔴
חובה מציאת ספק אמיתי
האפליקציה כרגע משתמשת במוצרים לדוגמה. עליך לפתוח חשבון אמיתי ב:
AliExpress Dropshipping Center – חינמי, קל להתחיל | CJ Dropshipping – משלוח מהיר יותר | Zendrop / Spocket – ספקים איכותיים לשוק המערבי
🔴
חובה הגדרת API keys אמיתיים
עדכן את קובץ .env עם מפתחות אמיתיים מהפלטפורמות שבחרת (ראה מדריך בלשונית "חיבור חנויות" באפליקציה).
🟠
חשוב חיבור API ספק אמיתי
כרגע המוצרים בחיפוש הם mock data. צריך לחבר ל-API של AliExpress / CJ Dropshipping / Zendrop.
AliExpress: open.aliexpress.com → AliExpress Dropshipping API
🟠
חשוב מסד נתונים אמיתי
כרגע המוצרים נשמרים בקובץ JSON וההזמנות ב-memory. בסביבת production צריך:
MongoDB Atlas (חינמי) או PostgreSQL + Supabase (חינמי) לשמירת הכל
🟠
חשוב Authentication – כניסת משתמשים
כרגע אין הגנת סיסמה. אם האפליקציה תהיה public צריך להוסיף login.
פתרון מהיר: NextAuth.js או Supabase Auth
🟠
חשוב Webhooks להזמנות אמיתיות
כרגע ההזמנות הן mock data. להזמנות אמיתיות מ-Shopify/eBay צריך להגדיר Webhooks שיקלטו הזמנות חדשות אוטומטית.
🟡
מומלץ הגדרת מחירים ריאלית
כנס להגדרות → שנה שקלול $→₪ לשקל הנוכחי, הגדר אחוז רווח ריאלי (עדיף 60-100% לדרופ שיפינג).
🟡
מומלץ שם עסקי + לוגו + תיאורי מוצרים
לפני שמפרסמים מוצרים: כתוב תיאורים ייחודיים (לא להעתיק מהספק), צלם/ערוך תמונות, וצור מיתוג עקבי.
🟡
מומלץ ביטוח ומסגרת משפטית
פתח עוסק פטור/מורשה, הגדר מדיניות החזרות ברורה, בדוק חבות מכס על יבוא לישראל מעל $75.

🌐 מעבר ל-Production

1

Build

cd frontend && npm run build   # מייצר תיקיית dist/
cd backend  && npm start        # מריץ production
2

Hosting מומלץ

Backend
Railway / Render / Fly.io – חינמי לגרסת hobby
Frontend
Vercel / Netlify – חינמי, מאוד פשוט לדפלוי
3

עדכן .env לפרודקשן

FRONTEND_URL=https://your-app.vercel.app
EBAY_REDIRECT_URI=https://your-backend.railway.app/api/ebay/callback
# כנ"ל לכל שאר ה-redirect URIs
⚠️ שנה את כל ה-Redirect URIs גם בלוחות הניהול של כל פלטפורמה!

🔧 פתרון בעיות נפוצות

?

הדף מוצג ריק / שגיאת 404

ודא ששני השרתים רצים: backend על פורט 3001 ו-frontend על 5173.

?

שגיאת CORS

ודא שב-.env הערך FRONTEND_URL=http://localhost:5173 תואם לכתובת ה-frontend.

?

OAuth לא מחזיר / redirect לא עובד

הבעיה נפוצה: ה-Redirect URI שרשמת בלוח הפלטפורמה חייב להיות זהה לחלוטין למה שב-.env. גם slash בסוף משנה!

?

המוצרים לא נשמרים בין הפעלות

המוצרים נשמרים בקובץ backend/data/products.json. ההזמנות (in-memory) לא נשמרות – זה מתוכנן כך בגרסה הנוכחית.

?

WooCommerce – "לא ניתן להתחבר"

בדוק: (1) ה-URL מתחיל בـhttps:// (2) ה-Consumer Key מתחיל בـck_ (3) פלאגין WooCommerce מעודכן (4) REST API מופעל בהגדרות WooCommerce
